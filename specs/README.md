# YX Specifications

This directory contains the specifications that define WHAT the YX protocol is and does.

## Specification Categories

### technical/
Core technical specifications:
- Protocol wire format
- Security mechanisms
- Transport layer
- Message handling

### functional/
Functional specifications (to be created):
- Use cases
- User workflows
- Integration patterns

### architecture/
Architectural decisions (to be created):
- Design decisions
- Trade-offs
- Rationale

### testing/
Testing specifications (to be created):
- Test plans
- Acceptance criteria
- Verification strategies

## Current Specifications

- `technical/yx-protocol-spec.md` - Complete UDP protocol specification

## Adding Specifications

New specifications should:
- Be written in markdown
- Define WHAT, not HOW
- Be implementation-agnostic
- Be traceable from code
